Ollama Troubleshooting Guide

Common Issues

1. Ollama Not Running

Problem: Ollama is installed but not running.

  • macOS/Linux: Run ollama serve in terminal
  • Windows: Ollama starts automatically — check the system tray
  • Verify with: ollama list

2. Port Already in Use

Problem: Port 11434 is in use by another application.

  • Check what's using it: lsof -i :11434 (macOS/Linux)
  • Stop the conflicting process, or set a custom port via Advanced Configuration

3. Model Not Downloaded

Problem: Ollama running but model not installed.

  • Download: ollama pull llama3.2
  • Verify: ollama list should show llama3.2
  • Embedding model (optional): ollama pull nomic-embed-text

4. CORS / 403 Forbidden

Problem: Ollama is reachable but blocking this extension's origin with 403.

Root cause: Ollama checks the Origin header. By default it only allows localhost origins, not moz-extension://.

Fix — these steps must be followed exactly:

  • Step 1: Stop Ollama — quit the menu bar app, or Ctrl+C the terminal running ollama serve
  • Step 2: In a new terminal, paste the following as ONE single command:
    OLLAMA_ORIGINS='moz-extension://*' ollama serve
  • Step 3: Keep that terminal open — closing it stops Ollama
  • Step 4: Click "Re-test Connection" in the extension

Common mistakes that DON'T work:

  • OLLAMA_ORIGINS='...' on one line, then ollama serve on the next — the env var is lost between commands
  • launchctl setenv OLLAMA_ORIGINS ... — only affects NEW launchd processes, not the current terminal session

Permanent fix (macOS/Linux):

echo 'export OLLAMA_ORIGINS="moz-extension://*"' >> ~/.zshrc && source ~/.zshrc

After this, plain ollama serve will always have the right origins.

5. Slow Performance

  • Use smaller model: ollama pull llama3.2:1b
  • First request loads the model (slow) — subsequent requests are fast

6. Remote Ollama Server

  • Use Advanced Configuration to set a custom endpoint, e.g. http://192.168.1.100:11434
  • Set OLLAMA_HOST=0.0.0.0 on the server side to allow remote connections

Test Commands

curl http://localhost:11434/api/version
ollama list
ollama serve
